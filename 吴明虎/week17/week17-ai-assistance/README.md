# 任务型多轮对话系统

一个基于Python的任务型多轮对话系统，支持多种场景的智能对话交互。

## 功能特性

### 核心功能
- **多场景支持**：支持"买衣服"、"看电影"等多个对话场景
- **意图识别**：使用文本相似度算法识别用户意图
- **槽位填充**：自动提取用户输入中的关键信息
- **多轮对话管理**：跟踪对话状态，自动追问缺失信息
- **Web界面**：提供用户友好的聊天界面

### 智能交互
- **困惑语句识别**：当用户输入"什么"、"怎么了"等困惑语句时，系统会提示当前场景和已选选项
- **非预期回复处理**：当用户输入不在预期范围内时，系统会提示当前可用的功能和说法示例
- **选项验证**：检查用户输入的选项是否在预设范围内，无效时提示有效选项
- **节点自由跳转**：同一层级的节点可以相互跳转，提供灵活的对话流程

## 技术架构

### 核心组件
- **DialogueSystem**：对话系统核心类
- **NLU (Natural Language Understanding)**：自然语言理解，负责意图识别和槽位填充
- **DST (Dialogue State Tracking)**：对话状态跟踪，管理对话状态
- **PM (Policy Management)**：策略管理，决定对话下一步操作
- **NLG (Natural Language Generation)**：自然语言生成，生成系统响应

### 技术栈
- **后端**：Python、Flask
- **前端**：HTML、CSS、JavaScript
- **数据存储**：JSON（场景配置）、Excel（槽位配置）

## 快速开始

### 安装依赖
```bash
pip install flask openpyxl pandas
```

### 启动服务
```bash
python web_chat_service.py
```

### 访问系统
打开浏览器，访问：http://127.0.0.1:8000/

## 使用说明

1. 在页面顶部选择一个场景（如"买衣服"或"看电影"）
2. 点击"开始新会话"按钮
3. 在输入框中输入您的需求，按Enter发送
4. 系统会根据您的输入进行响应

## 场景示例

### 买衣服场景
```
用户：我要买衣服
系统：请告诉我服装类型。
用户：衬衫
系统：请告诉我服装颜色。
用户：蓝色
系统：请告诉我服装尺寸。
用户：M
系统：为您推荐这一款，M号，蓝色衬衫...
```

### 看电影场景
```
用户：我要看电影
系统：请告诉我时间。
用户：8点
系统：请告诉我电影名称。
用户：阿凡达
系统：为你找到8点的阿凡达电影
用户：来个爆米花
系统：给你爆米花
用户：结束了
系统：好的，欢迎下次光临！
```

## 智能提示

### 困惑语句处理
当用户输入"什么"、"怎么了"、"不明白"等困惑语句时：
```
用户：什么？
系统：当前在买衣服场景，已经选择了：服装类型: 衬衫。当前问题：请告诉我服装颜色。
```

### 非预期回复处理
当用户输入不在预期范围内时：
```
用户：随便聊聊
系统：抱歉，我没太明白您的意思。目前您可以：开始看电影（可以说'我要看电影'）、买爆米花（可以说'来个爆米花'）、买可乐（可以说'来个可乐'）、结束观影（可以说'结束了'或'看完了'）。
```

### 选项验证
当用户输入无效选项时：
```
用户：紫色
系统：抱歉，'紫色' 不是有效的服装颜色选项。请从以下选项中选择：黑、白、红、蓝、绿、灰
```

## 扩展开发

### 添加新场景
1. 在`scenario/`目录下创建新的JSON配置文件
2. 定义节点、意图、槽位、响应和子节点
3. 重启服务即可使用新场景

### 场景配置格式
```json
[
  {
    "id": "node1",
    "intent": ["我要看电影"],
    "slot": ["#时间#", "#电影名称#"],
    "response": "为你找到#时间#点的#电影名称#电影",
    "childnode": ["node2", "node3", "node4"]
  }
]
```
